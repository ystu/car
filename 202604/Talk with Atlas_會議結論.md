# Talk with Atlas－會議結論

## 一、整體結論

本次兩份討論內容主要聚焦在保固案件的判定邏輯、維修佐證、打卡流程與人員能力養成。整體來看，目前保固案件並非完全不能通過，但流程紀律與判斷依據仍需強化，否則後續在原廠稽核或內部抽查時，仍會產生扣款與退件風險。

核心問題可歸納為三點：

1. 保固申請必須以「故障事實、因果關係、維修手冊」作為依據。
2. 打卡必須反映實際作業時間，不能事後補登或為了申報才補做。
3. 保固人員、技師、接待與主管之間需要建立一致流程與溝通機制。

## 二、保固判定原則

保固理賠不是以「想換什麼」或「預防性更換」為依據，而是要證明零件確實故障，並說明故障與更換項目之間的因果關係。

申請保固時應注意：

1. 必須說明故障原因、損壞部位及維修範圍。
2. 若主故障造成其他零件連帶損壞，需提出明確佐證。
3. 不得以「避免二次損壞」作為籠統理由，直接擴大更換範圍。
4. 不得將漏報項目拆成另一筆原因不明的案件補申請。
5. 送件內容應符合維修手冊、TPI 或原廠技術資料。

例如壓縮機案件中，乾燥包可用來判斷是否有鐵屑，但若冷排本身沒有損壞或沒有證據顯示鐵屑進入冷排，就不應直接申請更換冷排。保固申請應賠到合理範圍，不應矯枉過正。

## 三、維修佐證與測試方式

維修佐證必須能真實反映故障狀況。若測試方法不合理，即使車輛確實有異音或故障，也可能因佐證不足而被退件或扣款。

本次提到的軸承異音案件，主要問題如下：

1. 車輛架在頂高機上測試，無法模擬道路行駛時輪胎受力與摩擦狀況。
2. 異音判斷應以路試為主要依據，並提供更換前後同條件比較。
3. 測試儀器的頻道、線材顏色與錄影內容必須一致，否則容易被認定佐證不可信。
4. 若影片或資料顯示方式前後不一致，會產生造假或資料不完整的疑慮。

後續軸承、異音、底盤或動態測試案件，應優先採用路試與明確的更換前後比對，不建議以頂高機空轉作為主要佐證。

## 四、打卡與工單流程

打卡的目的不是單純給保固人員看，而是用來證明技師實際作業時間，並反映服務廠生產力、派工效率與流程管理狀況。

目前主要問題是，部分案件等到診斷完成、確認要申報什麼工時後，才開工單或補打卡。這種作法會造成診斷時間與打卡時間不一致，原廠會認定資料不具可信度。

後續應採取以下原則：

1. 預約車應在進場前或進場當下先建立工單。
2. 車輛進場並交給技師後，即可開始打卡。
3. 不需要等最終維修項目完全確定後才打卡。
4. 可先以一般檢修項目建立作業，確認實際故障後再調整申報內容。
5. 午休、待料、跨日維修或中途轉做其他車輛時，應使用暫停功能。
6. 同一名技師不得同時在兩台車上累計工時。

若技師已經去修另一台車，但前一台車的打卡仍持續計時，會造成工時失真。此問題不只影響保固申報，也可能影響客戶收費與內部管理判斷。

## 五、系統異常時的處理方式

若因系統異常導致無法打卡，必須留下手寫紀錄，並由服務經理或主管背書確認。

手寫紀錄應至少包含：

1. 工單號碼。
2. 技師姓名。
3. 實際開始與結束時間。
4. 異常原因。
5. 主管確認與簽名。

但此方式僅限於系統異常等少數例外狀況，不能變成常態作法。若多數案件都以手寫或事後補登方式處理，原廠仍可能不予採認。

## 六、工時、車型與維修手冊判斷

工時申報必須確認車型、引擎型式、配備與維修手冊內容。錯用工時代碼會造成高額扣款風險。

本次提到節溫器案件時，柴油車與汽油車工時差異很大，例如 EA288 / TDI 與 EA888 / TFSI 若判斷錯誤，工時可能相差數小時。這類案件應由技師先查詢，保固人員再覆核車輛規格與工時是否一致。

維修手冊中的必要更換項目也要正確解讀。有些項目是必須伴隨更換，有些只是選擇性或建議項目，不能全部視為必換。保固原則是合理理賠，不多賠、不少賠，也不為了保守而過度更換。

## 七、人員能力與內部溝通

保固人員需要同時理解原廠規範、維修手冊、技師說明與客戶服務需求，因此角色非常關鍵。若只依技師口頭說明送件，容易跟著錯誤判斷一起送出。

後續建議：

1. 保固人員應持續學習車輛產品、零件、工時與診斷檔判讀。
2. 技師應共同參與保固流程改善，不能只把送件責任交給保固人員。
3. 服務顧問、技師、保固人員應固定討論近期退件、扣款、TPI 或新問題。
4. 新進或非原廠體系出身技師，需加強原廠維修手冊與保固邏輯訓練。
5. 建議提供組織表，標註技師背景、訓練狀態及電動車課程資格。

## 八、外力與客訴案件判斷

部分案件可能涉及客戶服務壓力，但保固仍需依事實判定。若零件變形、破損或受傷位置明顯符合外力造成，應如實判斷，不應勉強以保固處理。

可依以下方式處理：

1. 若明顯外力造成，應由服務端向客戶說明，不宜送保固。
2. 若屬灰色地帶，應補強照片、檢查紀錄與判斷依據。
3. 若保固人員受到異常壓力，可透過 email 向原廠窗口說明疑慮。
4. 6 個月或 10,000 公里內的灰色案件可較彈性評估，但仍需排除外力。

## 九、改善要求

1. 建立預約車先開工單、進場即打卡的標準流程。
2. 明確規定打卡暫停使用時機，避免連續打卡造成工時失真。
3. 保固申請前必須確認故障事實、因果關係、維修手冊與工時代碼。
4. 異音與軸承類案件應以路試及更換前後比較作為主要佐證。
5. 系統異常無法打卡時，須採手寫紀錄並由主管背書。
6. 建立每週或每日短會，分享 TPI、退件案例、扣款原因與新問題。
7. 整理技師訓練狀態與電動車資格，作為派工與能力補強依據。
8. 建立保固人員與原廠窗口的詢問分級：不急案件用 email，急件可電話確認。

## 十、會議結論

兩次討論的共同結論是：目前問題不是單一案件處理錯誤，而是保固流程、技師作業紀律、佐證品質與跨部門溝通尚未完全建立標準。

公司後續應把重點放在流程制度化，而不是每次靠個案補救。只要能做到進場即開單打卡、維修依手冊、申報有佐證、工時有覆核、結案有責任歸屬，就能降低退件與扣款風險，也能讓服務廠的真實效率更容易被管理與改善。
