# 原廠保固稽核結果報告－會議結論

## 一、整體結論

本次原廠保固稽核結果整體表現尚可，缺失率與扣款金額經調整後約為 1.06%，預估扣款約 3.8 萬元。此次稽核屬首次輔導性質，標準已有適度放寬，因此目前結果不代表流程已完全合格。

本次最主要問題並非單一技術能力不足，而是保固作業流程、打卡紀律、工時判斷及維修佐證尚未制度化。後續若未改善，未來原廠或內部稽核標準恢復正常後，扣款風險會明顯增加。

## 二、主要缺失

### 1. 打卡時間與實際作業時間不一致

多數缺失集中在打卡時間與診斷、維修時間不一致。部分案件已完成維修，卻於隔日或數日後才補打卡，最嚴重有相差 5 天的情形。

事後補打卡無法證明技師實際作業時間，原廠會認定此紀錄不具參考價值，並可能列為扣款依據。

### 2. 診斷工時紀錄不足或與打卡不符

部分案件診斷檔、診斷工時與打卡紀錄對不起來。原廠抽到的案件與內部抽查結果一致，表示這不是單一偶發事件，而是目前流程中普遍存在的問題。

診斷階段常是保固判定的關鍵，若沒有即時打卡與紀錄，後續即使維修內容正確，也難以證明流程合理。

### 3. 工時代碼及車型配備判斷需加強

部分案件有工時代碼選錯或配備判斷不足的情形。例如汽油車與柴油車工時不同，EA888 與 EA288 若誤選，會造成工時差異過大並形成扣款風險。

雷達校正案件也顯示，申報前必須確認車輛實際配備與 PR code，不能只依工時項目直接申請。

### 4. 未完全依維修手冊與必要設定執行

部分案件涉及維修手冊要求或必要設定未落實，例如 J605 設定疑似選錯項目或未完成。此類問題若與後續零件損壞有因果關係，原廠可能判定理賠不合理。

另有安全相關項目，例如下護板螺絲應依規定更換，即使金額不高，也必須補強處理。

### 5. 結案與流程管理需改善

部分案件未即時結案或流程狀態未清楚關閉，容易造成後續稽核資料不完整。保固案件從進場、診斷、維修、交車到結案，應有明確責任分工。

## 三、改善要求

1. 預約車應於進場前或進場當下先建立工單，讓技師可於實際開始診斷時立即打卡。
2. 不得等診斷完成、確認更換項目後才開工單或補打卡。
3. 若尚未確認最終維修項目，可先建立一般檢修項目供技師打卡，後續再依實際維修內容調整申報項目。
4. 技師作業中斷、午休、待料或跨日維修時，應使用暫停功能，不得以單一連續時間或事後補登取代。
5. 保固申報前須確認車型、配備、PR code、維修手冊及正確工時代碼。
6. 必要設定、必要更換零件及安全項目須依維修手冊執行並留下佐證。
7. 保固申請不得以「預防性更換」作為理由，必須說明故障原因、損壞部位、維修範圍及因果關係。
8. 案件完成後應即時結案，避免稽核時出現流程未關閉或資料不完整。

## 四、後續追蹤重點

後續應優先追蹤以下四項：

1. 打卡是否於實際診斷、維修開始時即時執行。
2. 診斷工時與打卡時間是否一致。
3. 工時代碼、車型及配備判斷是否正確。
4. 維修手冊要求、必要設定及結案流程是否落實。

建議由服務主管、保固人員、接待及技師共同確認新流程，並定期抽查近期保固案件。若連續抽查未再發現重大異常，再逐步放寬人工攔截與事前審核。

## 五、會議結論

本次稽核結果可接受，但仍有明確改善空間。公司後續應將重點放在流程紀律與佐證完整性，而不是只針對單一案件補救。

打卡、診斷紀錄、工時判斷、維修手冊及結案流程必須形成固定作業標準。若能在近期內改善上述問題，後續保固案件通過率與稽核風險可望明顯降低。
