## 文件摘要

本服務專案適用於特定製造期間的 Caddy 車輛。冷卻水迴路中的泵浦啟動故障，可能導致冷卻水品質隨時間劣化；里程增加後，可能造成總成損壞。

針對受影響的 Caddy 車輛，提供 PV4RC 套件軟體。視車輛目前軟體版本而定，軟體更新可能包含額外功能與多項性能改良。

補救方式為：在受影響車輛上執行軟體套件更新。

## 作業流程

### 1. 檢查車輛是否受影響

若車輛於 `Vehicle-specific information`（個別車輛資訊）列入未維修完成名單，請檢查保固保養手冊是否已有 `P4CA performed`（P4CA 已執行）項目。

| 檢查結果 | 處理方式 |
| --- | --- |
| 出現事件故障項目 | 服務專案已執行，不需進一步作業。 |
| 未出現事件故障項目 | 依維修說明執行本服務專案。 |

作業範圍（規範）的檢查在 ElsaPro 內進行。

| 規範 | 名稱 |
| --- | --- |
| 01 | 軟體更新 Caddy 固定件 PV4 |

### 2. 重新程式化一般注意事項

- 務必使用最新的 ODIS 服務/GFS 版本。
- 軟體更新時，必須關閉所有不必要的電氣用品，例如通風、空調、座椅加熱、車內燈。
- 為求最高傳輸穩定性，建議診斷介面 VAS 6154 或 VAS 6154A（WiFi 診斷工具）僅在控制單元軟體更新的 USB 操作下使用。
- 請務必依照本文件的軟體更新順序進行。

### 3. 準備作業

#### ODIS - D3 和 EdgeBox

- 更新前，必須將 ODIS 更新至最新版本與最新的引導故障查修。
- 更新時資料會載入 EdgeBox；EdgeBox 必須有穩定網路連線。
- 若下載時出現問題，建議約 30 分鐘後重新開始下載。
- VCI 必須透過纜線連接到測試器。

#### 車輛

- 12V 車輛電瓶必須連接至電瓶充電器，電瓶 SOC 應至少為 90%。
- 整個更新過程中，讀取線圈中必須有 1 支車輛鑰匙。請將車輛鑰匙黏貼至讀取線圈。
- 若車輛鑰匙無法辨識，請遵循 TPI 2075259。
- 必要時，可關閉運輸/生產模式。
- 更新前必須正確完成 PDI。
- 若過去更換過控制單元，必須在軟體更新開始前執行啟動。
- 開始軟體更新前，必須刪除整個事件記憶。
- 打開引擎蓋並開啟危險警示燈。

### 4. 更新軟體

請先確認已完成「準備作業」與「重新程式化一般注意事項」。

1. 連接電瓶充電器並開啟。
2. 將車輛鑰匙（遙控器）放在讀取線圈上。
3. 開啟點火開關。
4. 將排檔桿置於 P 檔。
5. 將診斷測試器連線至車輛診斷連線與服務廠伺服器。
6. 配備第二個電瓶（零件號碼 8FV）的車輛，更新前及更新期間必須停用 `J934`（電瓶監控單元 2，接地電纜電瓶夾）或第二個電池。
7. 開始軟體更新前，務必執行點火循環。
8. 在 `diagnosis`（診斷）選擇 `start diagnosis`（開始診斷）並識別車輛。
9. 識別控制單元後，會自動開始引導故障查修。請依螢幕指示操作。
10. 在 `special functions`（特殊功能）模式下選擇 `update software`（更新軟體）。
11. 透過 `Update software with action code`（以作動代碼更新軟體）開始軟體更新。
12. 輸入作動代碼 `2K4R` 並確認。
13. 識別所有控制單元後，使用 `Finished/continue`（完成/繼續）開始軟體更新。
14. 所有控制單元會自動更新，請依螢幕指示操作。

> 重要：更新電子中央電系（第 3 項，診斷位址 0009）後，點火開關會關閉。必須在 10 秒內手動再次開啟點火開關，以便繼續後續軟體更新流程。

> 重要：更新駕駛輔助系統前攝影機（診斷位址 00A5）時，若軟體更新失敗，必須使用作動代碼 `2K4R` 重新啟動。

若每個控制單元名稱後皆有 `OK - correct`（OK - 正確），代表軟體更新成功。

> 注意：不可結束應用程式（診斷作業）。

軟體更新成功後，繼續執行 VKMS（車輛鑰匙管理系統）校正。

### 5. VKMS 校正

軟體更新後，必須執行 VKMS 校正。可經由緊急呼叫模組 `0075` 或儀表板 `0017` 執行。

1. 在「測試計畫」模式中選擇「選擇自我檢測」。
2. 選擇 `0075 VKMS functions -> calibrate VKMS` 或 `0017 VKMS functions -> calibrate VKMS`。
3. 將檢查附加至測試計劃。
4. 選擇 `Perform check...`（執行檢查...），並依說明執行測試程序。

可使用下列任一路徑：

- `Self-diagnosis systems -> 0075 - Emergency Call Module -> 0075 - VKMS functions -> Adapt VKMS`
- `Self-diagnosis systems -> 0017 - Instrument Cluster -> 0017 - VKMS functions -> Adapt VKMS`

若 VKMS 校正出現問題，請留意 TPI 2075259。

> 注意：不可結束應用程式（診斷作業）。

### 6. 基本設定與完成測試計畫

軟體套件更新可能導致部分基本設定或參數遺失。請依測試計劃完成車輛診斷測試器計算的檢查，並結束診斷作業；故障項目會自動刪除。

若無顯示軟體更新，表示軟體已為最新，無後續作業，接續「識別登記」。

與診斷測試器上的資訊不同，本專案不需產生就緒碼。

若重新程式化時出現問題，請按診斷測試器的協助與支援按鈕。若因診斷測試器當機而無法按下，請重新啟動測試器，開機後直接按協助與支援按鈕；前次會談資料仍可使用且可傳送。

## 問題處理

| 狀況 | 處理方式 |
| --- | --- |
| 點火開關未及時開啟 | 以作動代碼 `2K4R` 重新啟動軟體更新。 |
| 點火循環後無法辨識 VCI | 重新建立連線。軟體更新會自動繼續，或必須經由作動代碼 `2K4R` 重新觸發。 |
| ODIS 錯誤訊息 ERP 代碼 8088 或 8118 | 建立匯流排休眠，然後以作動代碼 `2K4R` 再次執行軟體更新。 |

接受的事件故障項目：

```text
DA 0009 - B200500 - 資料字串無效
```

## 客戶聯繫

本專案為使用客戶通知函的服務專案。

- 請通知受影響車輛客戶，最好以電話聯繫。
- 電話聯絡人必須記錄在車輛檔案內。
- 請使用「技術背景」章節的資訊向客戶說明。
- 確認所有受影響車輛已回服務廠檢查與維修。
- 客戶簽名前，請在工單上寫明已執行項目。
- 若車輛回廠時未執行本專案，請立即通知客戶本專案內容，可使用進口商的「儘管回廠但未執行專案」信件範本作為指南。
- 請通知公司新車與中古車部門，以便即時檢查受影響車輛，必要時進行修復。
